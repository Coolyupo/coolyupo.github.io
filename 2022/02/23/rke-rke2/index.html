RKE & RKE2 體驗日記

這篇只是一個homelab體驗日記(筆記)。

RKE

使用的環境就是：

os memory role ip
rke-operator ubuntu20.04 2G manual operator with rke binary 192.168.46.131
rke01 ubuntu20.04 6G control-plane、worker、etcd 192.168.46.132
rke02 ubuntu20.04 6G control-plane、worker、etcd 192.168.46.133
rke03 ubuntu20.04 6G control-plane、worker、etcd 192.168.46.134

node

因為rke是基於docker進行佈署的 每台node上先安裝docker，並設定預計登入的user在docker group當中(使其不需提權可以直接叫用docker cli)

開啟允許linux bridge呼叫iptables的系統參數

停用swap(還要再檢查一下/etc/fstab 有沒有取消swap的mount並免重開後又跑出來)

operator

安裝rke

設定到各台上的ssh憑證

透過rke config -p 互動式輸入產出預計佈署的cluster.yml

可以得到一個蠻完整的cluster.yml，有很多地方Kubernetes component能夠設定(也包含要去哪裡提取image)，所以可以參考官方添加自己想要的設定細節

接著執行

就可以看到開始佈署了

佈署完成後就可以看到多了兩個檔案，一個是使用rke佈署的cluster state、另一個則是kubectl使用的kubeconfig。

與kubeadm方式不同，在kube-system內沒有看到相關核心元件的static pod，東找西找了一下發現是在host上用docker run rancher/hyperkube:v1.22.6-rancher1 起來的(並指定network=host)

做一下inspect後可以看到entrypoint也有相關的參數設定，應該是根據一開始設定的cluster.yml有關

RKE2

使用的環境就是：

os memory role ip
rke01 ubuntu20.04 6G rke2 server nodes 192.168.46.132
rke02 ubuntu20.04 6G rke2 server nodes 192.168.46.133
rke03 ubuntu20.04 6G rke2 server nodes 192.168.46.134

2020年底kubernetes拋出將不再維護dockershim，並決定在1.24版時棄用，這也意味的現行很多CRI以docker為主的kubernetes在未來升級時會需要考量CRI的更換。

前陣子聽分享RKE2與RKE不同的地方，除了針對資安特化之外最大的差別我感覺應該是從原先的docker轉變直接使用containerd，所以今天就想說嘗試安裝體驗一下，裝起來比RKE的複雜一些(目前也還不是很懂詳細的配置應該怎麼操作，對這邊的文件有點閱讀障礙Orz)

接著在LAB開始前先提一下，根據官方的High Availability章節提到，我們需要先自行配置好Control plane的基本網路高可用條件，舉例可以從三個方向去著手

  1. L4 Loadbalancer
  2. RR DNS
  3. Virtual IP

這邊我就偷懶沒有去做(不然應該至少要以Keepalived+Nginx搭建)，取而代之後續的IP指向我都會直接指到rke01這台

1st server

我們需要先從一台server開始建起，依序官方的步驟執行shell

原本應該是應該手動建立/etc/rancher/rke2/config.yaml，去設定相關的token&tls-san，ex:

因為我忘了做這個，必須直接cat /var/lib/rancher/rke2/server/node-token

取得這個token

2nd&3rd server

一樣先執行shell並將service enable

以privileged user的身分創建/etc/rancher/rke2/config.yaml(可能需要創建目錄)

接著啟用，並觀察一陣子。

如果是單純的worker agent的話，則是在執行shell時需要去配置一些環境變數

final

rke2安裝的過程中有將相關的cli套件放置於/var/lib/rancher/rke2/bin/內 並且在安裝完後我們可以在/etc/rancher/rke2/rke2.yaml找到kubeconfig

然後稍微看一下這個簡單佈署完的長相(在任一台server皆可)......

與RKE不同，static pod又回來了XD

就先做到這好了，比起RKE很多設定我還搞不太清楚要怎麼弄XD，未來有機會的話應該會找時間再來詳細看看。

閒聊
